# 트럼프 카드 경마 게임 (Color Racing) 개발 계획안

## 1. 개요 및 목표
*   **게임의 목적:** 제비뽑기, 벌칙자 선정 등 추첨 (사다리 타기나 마블 룰렛을 대체할 수 있는 긴장감 높은 게임)
*   **핵심 컨셉:** 기존 트럼프 카드 경마 게임(`spec.md`)의 무작위 트랙 레이싱 룰을 기반으로 하되, 플레이어 수의 제한을 없애기 위해 카드 기호 대신 **'색상'**으로 유저를 구분합니다.
*   **타겟 환경:** 설치가 필요 없는 스태틱 웹 페이지 (GitHub Pages 배포 목적)
*   **디바이스 대응:** PC 및 모바일 화면 완벽 지원 (Responsive UI)

---

## 2. 상세 요구사항 및 핵심 기능

### 2.1. 게임 규칙 및 참여자 설정 (Setup)
*   **참여 말(유저) 설정:**
    *   최소 2인부터 4명 이상의 다수 유저 지원.
    *   각 유저는 고유의 매력적인 색상과 이름으로 구분됨 (예: 🔴 레드베어, 🔵 블루호크 등).
    *   **확률 조작(비율) 지원:** 유저마다 카드가 등장할 확률을 다르게 설정할 수 있음 (예: A말 카드 3장, B말 카드 1장으로 A의 당첨 확률 극대화).
*   **승리/당첨 조건 설정:**
    *   1등이 당첨(일반적 승리)
    *   꼴찌가 당첨(벌칙자 선정 시)
    *   사용자가 임의로 설정한 등수(N등)가 당첨되도록 유연하게 설정 가능.
*   **심화 규칙 옵션 (변수 창출):**
    *   `spec.md`에서 언급된 후퇴 기믹(절룩거림), 혹은 역전 기믹(조커 등장 시 1등 외 전원 전진) 등을 켜고 끌 수 있는 옵션 제공.

### 2.2. 게임 플레이 시뮬레이션 단계
*   **자동 플레이 시스템:** '시작' 버튼을 누르면 유저의 추가 조작 없이 자동으로 턴이 진행되며 시뮬레이션 됨.
*   **시각적 연출 (Visuals):**
    *   카드가 뒤집히는 다이내믹한 3D 애니메이션이나 부드러운 전환 효과(Transition)로 카드 게임 특유의 감성 요소 유지.
    *   트랙 위를 달리는 색상들의 이동을 매끄럽게 표현.
*   **현재 순위 마커 (Rankings):**
    *   화면 한편에 1등부터 최하위까지의 실시간 순위를 직관적으로 표시하며, 순위 역전 시 부드러운 애니메이션으로 스왑(Swap)됨.
*   **실시간 경마 중계 텍스트 (Casting):**
    *   단순히 턴의 결과만 딱딱하게 텍스트로 보여주는 것이 아니라, 턴 사이클에 맞춰 **"자! 게임 시작됩니다!"**, **"무서운 기세로 치고 나오는 파란 말!"**, **"빨간 말, 뼈아픈 실책으로 뒤쳐집니다!"** 와 같은 경마 감성 대사 무작위 출력.
    *   텍스트는 타이핑 효과나 페이드인/아웃을 사용해 끊김 없이 아나운서가 실제 중계하는 듯한 자연스러운 흐름을 연출.

### 2.3. 결과 출력 단계
*   게임이 끝나는 즉시 미리 설정한 승리 규칙(예: 3등 당첨)에 부합하는 유저를 화려한 애니메이션(폭죽 폭발, 팡파레 등)과 함께 명확하게 하이라이트.

## 3. 기술 스택 및 아키텍처 (Tech Stack)

### 3.1. Front-end
*   **언어:** HTML5, CSS3, Vanilla JavaScript (또는 Vite 기반의 경량화된 바닐라 프로젝트 구성으로 번들링 성능 향상)
*   **디자인 (Styling):**
    *   Vanilla CSS 만으로 현대적인 느낌(Modern Aesthetics) 구현 (Glassmorphism, 매끄러운 그라데이션, 화려한 색상 팔레트 사용).
    *   모바일에서는 트랙이 세로형이나 스크롤형으로 자동 조정되도록 Flex/Grid를 활용한 반응형 설계.
*   **애니메이션 기법:**
    *   카드 뒤집힘: CSS `transform: rotateY()` 활용.
    *   말 이동 및 순위표 변경: CSS `transition` 및 브라우저 최적화된 `transform` 이동. 중계 텍스트는 `requestAnimationFrame` 혹은 CSS Keyframes 조합.

### 3.2. 논리 모델 구조
*   `Deck` 모델: 설정된 확률 수치에 비례하도록 카드를 배열에 추가한 뒤, 완벽한 셔플(Fisher-Yates) 방식으로 무작위성 확보.
*   `Track` 모델: 경주로의 길이와 칸 상태, 장애물(변형 규칙용)을 관리.
*   `GameEngine` 컨트롤러: `setInterval`이나 루프를 통해 게임의 턴 과정을 통제하고 UI 업데이트 및 현재 등수를 실시간으로 계산(도착 점수 + 트랙 내 진행 위치 기준).

---

## 4. 구현 단계 (Milestones)

*   **[Step 1] 프로젝트 생성 및 기본 마크업:** Vite 프로젝트 세팅, HTML/CSS 아웃라인(설정 화면, 게임 화면, 결과 화면) 마크업, CSS 디자인 시스템 기초(컬러 변수 및 애니메이션 준비).
*   **[Step 2] 게임 로직 개발:** 유저 입력(이름/확률)에 기반한 동적 Deck 생성, 셔플, 말 객체화 및 등수 계산/판정 알고리즘 작업.
*   **[Step 3] 자동 시뮬레이션 및 연출 결합:** 턴의 진행에 따라 실제 DOM 요소(말, 카드 뒤집기 애니메이션)가 부드럽게 연동되게 하고 실시간 순위 변동 표현.
*   **[Step 4] 경마 중계 시스템:** 턴 시작/중간/종료/역전과 같은 특정 컨텍스트 이벤트 상황을 감지해 적절한 경마 캐스터 텍스트를 자연스럽게 렌더링.
*   **[Step 5] 최종 최적화 및 배포:** PC/모바일 교차 테스트, UI 디테일(모바일 비율 조정) 향상 및 GitHub Pages 환경을 고려한 정적 웹 배포 빌드.
